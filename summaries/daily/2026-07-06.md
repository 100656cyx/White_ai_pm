## 工具使用体感

今天的学习重点不是直接使用某个 AI 工具，而是理解 Agent 工具调用背后的框架机制。

最重要的体感是：Tool Calling 并不是模型真的去执行外部工具，而是模型输出结构化的工具调用意图，再由本地 Agent 程序解析并执行。这个边界非常关键，因为它决定了权限控制、错误处理、用户确认和审计应该放在哪里。

## 产品经理视角收获

今天最大的产品收获是：Agent 产品不能只看模型能力，还要看工具设计和流程编排。

ReAct 更像一个不断调用工具直到任务完成的循环，适合解释和实现基础 Agent。Reflection 在 ReAct 基础上加入负反馈，让模型检查工具执行结果是否满足原始任务。Replan and Execute 更适合复杂任务，因为它允许先制定整体计划，再根据执行结果调整计划。

从 AI 产品经理视角看，不同运行模式会影响：

- 任务成功率
- 用户可解释性
- 执行成本
- 延迟
- 失败后的恢复能力
- 用户是否信任 Agent 的动作
