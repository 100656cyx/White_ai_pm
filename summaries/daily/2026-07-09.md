## 关键收获

Agent 从 Demo 走向产品，需要的不只是更强模型，还包括一套可靠的运行系统。

关键工程能力的产品价值：

- Session：让任务可以保存、恢复和追踪。
- Memory：让长期偏好和稳定事实可以跨任务复用。
- Channels：让 Agent 接入用户真实工作入口。
- Cron：让 Agent 支持定时和周期性任务。
- Permission：让高风险动作进入人工确认。
- Sandbox：限制 Agent 的影响范围。
- Plugin：让平台能力可以扩展。
- Multi-Agent：让复杂任务可以分工处理。

## 产品判断

一个能产品化的 Agent，需要同时具备：

- 任务连续性。
- 操作可控性。
- 风险边界。
- 失败恢复能力。
- 可扩展工具生态。
- 清晰的用户反馈和审计记录。

模型能力决定 Agent 的上限，工程系统决定它能不能被稳定、安全、长期地使用。
