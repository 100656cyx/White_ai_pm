## 工具使用体感

今天的重点不是使用某个现成 AI 工具，而是理解 Agent 工具调用背后的实现逻辑。

一个关键收获是：大模型本身并不会直接调用工具。模型只会生成工具调用意图，真正执行工具的是 Agent 框架中的工具调用组件。因此工具权限、用户确认、错误处理和执行结果回传，都应该作为 Agent 产品设计的一部分。

## 产品经理视角收获

ReAct 的优势是流程清晰、可解释性强，适合帮助用户理解 Agent 如何一步步完成任务。但它也容易陷入局部判断，对复杂任务缺少整体规划。

Plan and Execute 更适合复杂任务：先规划整体 Todo，再执行，并在过程中根据结果重新规划。对于 AI 产品经理来说，这两种模式的差异会直接影响产品体验、任务成功率、用户信任和可控性。
